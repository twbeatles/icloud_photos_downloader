## 주요 잠재 이슈 (심각도 순)

### 1) [높음] `Need MFA` 상태가 인증 완료 후에도 장시간 고정될 수 있음
- 근거 코드:
  - `app/core/runner.py:253-261`
  - `app/core/log_parser.py:97-103`
- 설명:
  - MFA 감지 시 상태를 `NEED_MFA`로 바꾸는 로직은 있으나, 이후 인증 완료를 감지해 `RUNNING`으로 되돌리는 로직이 없습니다.
  - 특히 `watch` 모드처럼 장시간 실행 시 상태 배지가 계속 `Need MFA`로 남아 오해를 유발할 수 있습니다.
- 영향:
  - 실제 실행 중인데도 사용자에게 "인증 대기"로 보이는 상태 왜곡 가능
- 제안:
  - MFA 이후 진행 신호(다운로드 라인, webui 완료 라인 등)를 감지하면 `RUNNING` 복귀 처리 추가
  - 상태 전이 테스트 케이스 추가

### 2) [높음] 에러 판정 정규식이 넓어 오탐 시 최종 상태가 `ERROR`로 고정될 위험
- 근거 코드:
  - `app/core/log_parser.py:22-24`, `84-87`, `111-116`
- 설명:
  - 현재 에러 판정은 `\berror\b` 단일 키워드 기반입니다.
  - 로그에 "error" 단어가 정보성/부가 문맥으로 등장해도 `error_count`가 증가하고, 종료 코드가 0이어도 최종 상태가 `ERROR`가 됩니다.
- 영향:
  - 정상 종료 케이스가 실패로 표시될 가능성
  - 자동 재시도/히스토리 정확도 저하
- 제안:
  - 로그 레벨(prefix) 기반 판정 또는 패턴 세분화
  - 실제 `icloudpd` 샘플 로그 기반 파서 회귀 테스트 추가

### 3) [중간] `QProcess` 대기 호출이 UI 스레드를 블로킹함
- 근거 코드:
  - `app/core/runner.py:189` (`waitForStarted(5000)`)
  - `app/core/runner.py:203-205` (`waitForFinished`)
- 설명:
  - 시작/중지 시 최대 수 초간 UI 이벤트 루프가 멈출 수 있습니다.
- 영향:
  - 앱이 일시적으로 멈춘 것처럼 보이는 UX 저하
- 제안:
  - 비동기 시그널 중심으로 전환하고 blocking wait 제거
  - 중지 시 타임아웃 경과 상태를 UI에 별도 표기

### 4) [중간] 문서 요구와 UI 동작 불일치: `watch` 모드에서 자동 재시도 "비활성"이 UI에 반영되지 않음
- 근거 문서:
  - `README.md:20` (watch 모드에서는 자동 재시도 비활성)
- 근거 코드:
  - `app/ui/settings_view.py:315-317` (watch 토글 시 interval만 제어)
  - `app/ui/settings_view.py:343-347` (auto-retry는 별도 활성)
  - `app/ui/main_window.py:295-296` (실행 시에만 watch면 retry 억제)
- 설명:
  - 현재는 실행 로직에서만 재시도를 막고, 설정 UI에서는 사용 가능하게 보여줍니다.
- 영향:
  - 사용자 혼동 (설정은 켜져 있는데 실제로 동작하지 않음)
- 제안:
  - watch ON 시 auto-retry 체크박스/파라미터 비활성 + 안내 문구 표시

### 5) [중간] 재시도 대기 상태에서 사용자가 "대기 중 재시도"를 명시적으로 취소하기 어려움
- 근거 코드:
  - `app/ui/main_window.py:301-313`, `314-317`
  - `app/ui/run_view.py:150-152` (ERROR 상태에서 Stop 비활성)
- 설명:
  - 재시도 타이머가 설정되면 실제 프로세스는 멈춰 있어 Stop 버튼이 비활성입니다.
  - 사용자는 타이머 취소를 위한 전용 조작을 찾기 어렵습니다.
- 영향:
  - 잘못된 재시도 대기 지속 가능성, UX 혼란
- 제안:
  - `retry pending` 상태/카운트다운 표기
  - "Cancel Retry" 버튼 제공 (타이머 stop)

### 6) [낮음] Windows에서 외부 실행 파일 유효성 검사가 느슨함
- 근거 코드:
  - `app/core/runner.py:36-41`
- 설명:
  - Windows에서는 "존재하는 파일"이면 실행 가능 후보로 간주합니다.
  - 실행 불가능 파일 선택 시 결국 시작 실패 메시지로만 드러납니다.
- 영향:
  - 설정 오류를 늦게 인지
- 제안:
  - 확장자(.exe/.bat/.cmd 등) 또는 실제 실행 가능성 검증 강화
  - 설정 단계에서 즉시 경고
